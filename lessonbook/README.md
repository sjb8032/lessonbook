# 레슨북

레슨 예약 · 회차 · 일지를 한 곳에서. 카톡으로 "이번 주 언제 되세요?" 묻지 않는 레슨 관리.

> 이름은 임시입니다. `src/app/layout.tsx`, `AppNav.tsx`, `login/page.tsx`에서 바꾸면 돼요.

## 기능 (v0 + v1)

**선생님**
- 주간 시간표에서 빈 시간 탭 → 예약 가능 시간으로 공개
- "이번 주 시간표를 다음 주로 복사" 한 번으로 반복 스케줄 관리
- 수업 완료 처리 → 회차 자동 카운트, 사이클(기본 4회) 끝나면 학생에게 결제 알림 자동 발송
- 학생 목록: 완료 회차 / 결제분 / 잔여를 한눈에, 미납은 빨간펜 표시
- 학생 상세: 입금 확인, 결제 내역, 레슨 일지(진도/비고/과제), 비공개 메모
- 6자리 가입 코드로 학생 연결 (링크·앱 설치 강요 없음)

**수강생**
- 선생님 시간표에서 빈 시간 바로 예약 / 12시간 전까지 셀프 취소 (취소하면 그 시간이 다시 열려 다른 학생이 잡을 수 있음)
- **시간 교환**: 다른 수강생의 시간을 탭 → 내 수업과 바꿔달라고 요청 → 상대가 수락하면 원자적으로 교환 + 선생님께 자동 통보 (이름은 마스킹됨)
- 내 수강 현황: 완료 회차 / 결제분, 사이클 종료 시 결제 안내(계좌·송금 링크)
- 선생님이 쓴 레슨 일지 열람

## 셋업 (10분)

1. **Supabase 프로젝트 생성** — [supabase.com](https://supabase.com) 무료 플랜이면 충분
2. **스키마 적용** — Supabase 대시보드 → SQL Editor → `supabase/migrations/0001_init.sql` 전체 붙여넣고 Run
3. **환경 변수** — `.env.example`을 `.env.local`로 복사, 대시보드 Settings → API의 URL과 anon key 입력
4. **실행**
   ```bash
   npm install
   npm run dev
   ```
5. **배포** — Vercel에 연결하고 환경 변수 두 개만 넣으면 끝. 배포 후 Supabase 대시보드 → Authentication → URL Configuration에서 Site URL을 배포 주소로 변경 (매직링크가 그 주소로 리다이렉트됨)

## 사용 흐름

1. 선생님이 먼저 가입 (이메일 매직링크) → 설정에서 결제 주기·수강료·계좌 입력
2. 스케줄 상단의 **가입 코드 6자리**를 학생에게 공유
3. 학생 가입 시 코드 입력 → 자동 연결 → 바로 예약 가능
4. 수업 끝나면 선생님이 슬롯 탭 → "수업 완료 처리" → 회차 +1, 4회차마다 결제 알림 자동

## 아키텍처 노트

- Next.js 15 (App Router) + Supabase (Postgres + Auth + RLS)
- 예약/취소/스왑/완료처리는 전부 **Postgres security definer 함수**로 원자 처리 — 동시 예약 경합, 스왑 도중 취소 같은 레이스 컨디션을 DB 레벨에서 차단
- 다른 수강생 이름은 서버(DB 함수)에서 마스킹되어 내려옴 — 클라이언트에 원본이 노출되지 않음
- 알림은 인앱(🔔) 테이블 기반. 카카오 알림톡은 v2에서 확장 지점

## 의도적으로 뺀 것 (스코프)

- **PG 결제 없음** — 결제는 "알림 + 계좌/송금 링크 + 선생님 입금 확인" 구조. 돈을 직접 만지지 않아 전자금융업 이슈가 없음. PG(토스페이먼츠/포트원 하위몰 정산)는 v2
- **카카오 로그인** — Supabase Auth에서 Kakao provider 켜면 코드 몇 줄로 추가 가능. v0는 설정이 필요 없는 이메일 매직링크
- **60분 고정 그리드** — `teacher_settings.lesson_minutes` 컬럼은 있으나 UI는 정시 시작 60분 기준. 30분 레슨은 다음 반복에서
- **1:1 관계 가정** — 학생 화면은 첫 번째 선생님 기준. 다중 선생님 데이터 모델은 이미 지원(enrollments), UI만 확장하면 됨
